Hook Firing Order

  • UserPromptSubmit → 5 unified hooks (intent, memory, profile, queue, context)
  • PreToolUse → 18 agent-scoped guards
  • PostToolUse → output capture + validation
  • session-end → HMAC POST to HQ API

Skill Loading Strategy

  • Skill SKILL.md: ~300 tokens (always, on agent start)
  • rules/ files: ~2-5KB each (on demand, when relevant)
  • references/ files: unlimited (only when deep info needed)
  • scripts/ templates: on explicit request

Agent Selection Logic

  • CC reads task intent from UserPromptSubmit hooks
  • CC sees all 38 agents in manifest
  • Selects 1-3 best-fit agents per task domain
  • Uses Task tool with subagent_type for routing

Key Invariants

  • Skills are never user commands (except 17 /ork: ones)
  • CC is always the orchestrator, never a direct executor
  • Hooks fire at defined lifecycle points, not ad hoc
  • Memory persists via knowledge graph across sessions
