Major features shipped in v7.0.0

The headline: 3 plugins became 1. But the branch grew into 7 major initiatives.
CORE Unified Plugin Architecture ▶

Before: 3 plugins (ork, orkl, ork-creative) with overlapping skills, separate manifests, confusing install story.

After: 1 plugin (ork) with 2 thin aliases (orkl, ork-creative) that redirect. All 69 skills, 38 agents, 36 hooks in one place.

Commits: 7ac9aebe, 225b833f, fac68047, 45b87da3, 02dc5ee5

Haiku 4.5 scores 90.8% on PinchBench — nearly matching Opus 4.6 (90.6%). Shifted 10 data-driven agents from Sonnet to Haiku.

Savings: $1,478/year, 12.7% cost reduction per session.

4 breaking changes in this release

All are intentional architecture changes. No behavioral regressions expected for end users, but plugins installed from v6.x need a fresh install.
BREAKING Plugin unification — 3 plugins → 1 + 2 aliases 225b833f ▶

What changed: orkl and ork-creative are now thin alias redirects to ork. All 69 skills live in one plugin.

Migration: Uninstall old plugins, install ork. Alias names still work for backward compat.

BREAKING Token diet — skill aliases removed, AGENTS.md deleted, 12 skills demoted 3db94678 ▶

What changed: Skill aliases (shortcuts) removed. The standalone AGENTS.md routing table deleted. 12 low-frequency skills demoted from user-invocable to internal-only.

Impact: Reduced per-session token overhead. Skills still work, just not listed in prompt. Call them directly by name.

BREAKING CC capability audit — 9 structural changes 71ae2a18 ▶

What changed: Added maxTurns to agents, explicit timeout tuning for hooks, worktree isolation for team-spawning agents, expanded settings.json with keybindings/respectGitignore/plansDirectory/spinnerVerbs.

Impact: Agents now have turn limits (prevents runaway cost). Hooks have tighter timeouts.

BREAKING Version bump to 7.0.0 225b833f ▶

Minimum Claude Code version: >= 2.1.59 (was 2.1.56)

Uses ConfigChange hook event, HTTP hook type, and agent-scoped hooks — all require 2.1.59+.

3 bugs fixed, 3 still open

Context over-injection is the #1 issue class. Session start overhead dropped from ~8,121 to ~5,500 tokens but still above 5,000 budget.

Still Open
BUG memory-context keyword gate fires on ~70% of prompts #865 ▶

Trigger keywords include "add", "fix", "create", "build" — matches virtually every dev prompt. Needs semantic intent detection or 2+ keyword requirement.

BUG realtime-sync fires on "must"/"cannot" in any Bash output PostToolUse ▶

"npm install" output with "must be installed as peer dependency" triggers "[IMMEDIATE SYNC] Critical decision detected..." — 120-150 wasted tokens.

BUG Cross-project context injects [SESSION-END] noise every turn #865 ▶

~40-80 tokens of "[SESSION-END]" timestamps injected. The gating catches empty output but not low-value output. Never been useful in 38 sessions.

22 brainstormed ideas across 3 design proposals

From the NotebookLM-powered brainstorming session. Organized into 7 clusters. None have started implementation — these are the v7.1+ roadmap.
The Three Designs

A: The Bridge Layer

Effort: Medium Risk: Low ~8 days
  • Stack profile file (.claude/stack-profile.json)
  • CC infrastructure scan (Phase 1b)
  • Context packs per agent type
  • Living agent brief from frontmatter

Recommended first. No dependencies.

B: Structured Handoffs

Effort: High Risk: Medium ~10 days
  • Typed artifact blackboard (.claude/blackboard/)
  • Agent provides/requires manifests
  • Agent handoff bus
  • Session knowledge graph persistence

Depends on Design A.

C: Skill Pipelines + Intelligence

Effort: High Risk: Medium ~12 days
  • Pipeline YAML loader
  • Model velocity calibrator
  • Error pattern router
  • Skill chain anticipator
  • Quality regression detector

Depends on Design A + B.
