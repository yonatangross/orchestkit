# Rule Categories

## 1. Five Whys Technique (5-whys) — HIGH — 1 rule

Iterative causal analysis for linear problems. Drills from symptom to root cause in 5 steps.

- `rca-five-whys.md` — 5 Whys process, examples, when it falls short

## 2. Fishbone Diagram (fishbone) — MEDIUM — 1 rule

Multi-factor root cause visualization using category-based brainstorming. Best for problems with multiple contributing factors.

- `rca-fishbone.md` — Ishikawa diagram, software categories, structured analysis process

## 3. Fault Tree Analysis (fault-tree) — MEDIUM — 1 rule

Top-down deductive analysis for critical systems. Maps all paths to failure using boolean logic.

- `rca-fault-tree.md` — AND/OR gates, FTA symbols, when to use for safety-critical systems
