Priority Gaps to Close

P0: Interaction Commands (15 missing)
Only click, fill, press documented. Missing: type, dblclick, focus, hover, check/uncheck, select, scroll, scrollintoview, drag, upload, keyboard type/inserttext, keydown/keyup, click --new-tab. This is the CORE of browser automation and we document 17% of it.
P0: Storage API (0/5 = zero coverage)
localStorage and sessionStorage commands are entirely absent. No agent or skill documents storage local, storage local set, storage session. Critical for SPA testing, token management, feature flag manipulation.
P1: Screenshot --annotate (visual debugging)
screenshot --annotate adds numbered labels to elements — extremely useful for a11y audits and visual debugging. Not documented anywhere.
P1: cookies get/clear (read + cleanup)
We document cookies set but not reading (cookies) or clearing (cookies clear). Can't debug auth issues without reading cookies.
P1: --confirm-interactive (human-in-the-loop)
v0.15 supports terminal-based human confirmation prompts. Not documented — relevant for headed mode flows and admin panel automation.
P2: eval -b / --stdin (base64 + piped JS)
Advanced eval modes for large scripts. Low usage frequency but useful for complex extraction.
P2: diff url --selector (targeted comparison)
Compare specific page sections between URLs. Useful but niche.

Honest Verdict: 4.5/10

We told ourselves 93% adoption. The real number is 45%. Here's why:

  • The "93%" was cherry-picked. We counted 28 "major" commands and matched 26. But agent-browser has 67 distinct commands/flags. We have 30.
  • Interaction is the core — and we flunked it. 15 of 18 interaction commands are undocumented. scroll, hover, select, check, drag, upload — none of these exist in any agent. That's the bread and butter of browser automation.
  • Storage API = zero. An entire feature category (localStorage, sessionStorage) is completely absent. No agent, no skill, no rule mentions it.
  • The new stuff (diff, network, state, confirm) is solid. The #947/#948/#949 sprint delivered well on v0.13-v0.15 features. That's genuinely good work.
  • Hook enforcement is excellent. 5/5 checks, env-configurable, proper SSRF prevention, rate limiting, robots.txt. This is the strongest part.
  • Agent spread is thin. Only 7/38 agents document agent-browser at all. Of those, only web-research-analyst has decent coverage (62%). The other 6 are tokenistic.
