PR playground · feat/yonyon-brand-keying-orank-discovery · how a crawler/LLM resolves a "yonyon"-keyed query, before vs. after this change.

The problem this PR addresses. orank's Discovery scan keys the brand off the apex domain yonyon and searches for "yonyon" — which (a) collides with an unrelated musician and (b) misses every developer resource keyed to "OrchestKit". This PR gives a "yonyon"-keyed agent a canonical, disambiguated resolution target (a /yonyon page + a first-class Yonyon Organization entity) without claiming any identity we don't hold.

Disambiguation FAQ shipped on /yonyon

    
Is Yonyon (OrchestKit) related to the musician of the same name?
      
No. On this site, Yonyon is an independent software studio — the publisher of OrchestKit, a free, open-source plugin for Claude Code. It is not affiliated with the DJ/music producer who also uses the name.

    
What does Yonyon make?
      
Developer tooling for AI-assisted engineering. Its flagship product is OrchestKit — 111 skills, 37 agents, and 210 lifecycle hooks for Claude Code, MIT-licensed and free.

    
Where are Yonyon's developer resources?
      
All under orchestkit.yonyon.ai: the docs, an OpenAPI spec, an MCP server, an /ask endpoint, llms.txt, and well-known agent/API discovery files. Source on GitHub.
