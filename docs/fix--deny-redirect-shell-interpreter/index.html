The block kept happening. Here is why, and the fix.

OrchestKit · pretool/bash/dangerous-command-blocker · PR #3121

curl <url> | python3 is DENIED because remote bytes into an interpreter is the remote-code-execution shape. That is correct. It recurred because the deny reason said "blocked, dangerous" without saying what to do instead, and Claude Code keeps no memory of a denied command, so the model re-emitted the same idiom forever. This PR makes the deny redirect. Pick a command below and toggle the old vs new message.

🛡️ Guard verdict

    📨 Reason fed back to the model

    Colour key in the command: network source (curl/wget/http) · | real pipe · interpreter target (sh/bash/python3/node) · safe target. A DENY needs a real pipe into an interpreter; a || is logical OR, not a pipe; and local data into an interpreter is allowed (only a network source makes it RCE, #3096).
