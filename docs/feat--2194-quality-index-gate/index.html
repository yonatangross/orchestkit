Quality Index Gate / when does a regression trip CI?

Each component has a committed baseline score. The gate re-aggregates the current results and fails when a component's score drops by more than the threshold, or disappears entirely. Drag a component's current score down, or drop it, and watch the gate flip. This mirrors check-eval-regression.sh exactly (default threshold 0.05).

PASSED

No component regressed beyond the threshold.

The gate this models

# aggregate current results into results/quality-index.json
bash scripts/eval/aggregate-quality-index.sh
# fail (exit 1) if any baseline component drops > threshold or disappears
EVAL_REGRESSION_THRESHOLD=0.05 bash tests/evals/scripts/check-eval-regression.sh
# accept an intentional change
bash tests/evals/scripts/check-eval-regression.sh --update-baseline
