Why This Matters

Before CC 2.1.101, context: fork and agent: in skill frontmatter were silently ignored. Every skill ran in the parent context, and agent delegation never happened. Now they're enforced. This means:

71 skills with context:fork will now actually run in isolated fork contexts — they won't see parent conversation context.
51 skills with agent: will delegate to the specified agent type for loading.

Without this audit, 26 of those assignments were wrong — skills would break silently when users upgrade to CC 2.1.101.
