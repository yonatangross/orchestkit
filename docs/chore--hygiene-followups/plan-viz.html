🔁 Recurrence guard

The PR-Playground gate forces a new docs/<branch>/*.html onto main every PR. Backlog was cleared in #2841 — this stops silent regrowth.

monthly cron ──► scan (read-only) │ drift > 0 ? ┌───┴───┐ yes no │ └─ exit quiet ▼ open/update tracking issue │ you run apply ──► archive + removal PR (human-reviewed)

Why the schedule mutates nothing

# the scheduled job is READ-ONLY by construction
schedule   → drift-scan job   permissions: contents: read, issues: write
           → scans, opens an issue if drift. No files, no branches touched.

dispatch   → prune job        permissions: contents: write
           → apply is opt-in AND lands via a PR, never a push to main
           → cross-branch mutation stays operator-gated (can't pre-test in CI)

Guards on the prune logic

only touches   docs/<type>--<slug>/ dirs
skips if        a matching branch still exists on origin (PR not merged)
age gate        newest commit older than 30 days
KEEP set        9 dirs hard-excluded (audits, site, feat--activation-audit, …)
validated       shellcheck ✅  actionlint ✅  live scan ✅ (0 prunable now)
