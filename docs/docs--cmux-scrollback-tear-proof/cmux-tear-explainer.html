⚙️ המנגנון — למה זה קורה

ב-cmux 0.64.20 נוספה תכונה חדשה (PR #7758, דלוקה כברירת מחדל): דחיסת scrollback. היא דוחסת דפי היסטוריה "קרים" ומחזירה את הזיכרון הפיזי שלהם למערכת ההפעלה.

📄 דף היסטוריה מתקרר → נדחס (LZ4)
↓ הזיכרון הפיזי מוחזר ל-OS (MADV_FREE_REUSABLE)
✅ קריאה בטוחה: דרך Node.page() — משחזרת קודם
↓ אבל…
❌ שלושה מקומות עוקפים: pageAssumeResident()
↓ קוראים זיכרון שכבר הוחזר ל-OS
🧵 בייטים ישנים + מה שתפס את הזיכרון = שורה קרועה

הזיכרון של דפי טרמינל אורז שורות רצוף, ולכן התערובת נשזרת ברמת התו. זה בדיוק מה שראית.

Compression decommits page memory; three write paths bypass the restore via pageAssumeResident() and read decommitted memory → character-level tear.

לא ניחוש. הקוד של cmux מצהיר דבר אחד במקום אחד, וההיפך במקום שני:

Screen.zig:895 — מצדיק את העקיפה"…grow does not compress pages, so the node must still be resident here."

בנוסף: timing גרסאות 0.64.14/15 (לפני #7758) רצו חודשים נקי; התקלה חדשה ב-0.64.20. הבעלים של cmux עצמו הצביע על #7758 ב-issue #8229.

Self-contradicting comments in the same file + a clean→broken version window pinned exactly to when #7758 landed. Owner already fingered #7758 in #8229.

🔧 "toolchain mismatch" — למה לא הרצנו את הבדיקה כאן

כתבתי בדיקה אוטומטית שאמורה להיכשל על הקוד הפגום ולהוכיח את הקריעה. אבל לא הצלחתי להריץ אותה על המק שלך. הסיבה פשוטה, ולא קשורה לבאג:

🔨 כדי לבנות את cmux צריך כלי בשם zig, גרסה 0.15.2 (זה מה ש-cmux דורש)
↕ אי-התאמה
🍎 המק שלך על macOS 26.5 — חדש יותר מ-zig 0.15.2

zig 0.15.2 לא מכיר את הספריות של macOS 26, אז השלב האחרון של הבנייה (קישור) נכשל. זו אי-התאמה בין גיל הכלי לגיל מערכת ההפעלה — לא שום דבר בקוד עצמו. בדיקה טריוויאלית כן עברה, מה שמאשר שהכלי תקין בעיקרון.

המקום הנכון להריץ: ה-CI של cmux עצמו (לינוקס + הגרסה הנכונה של zig), או קונטיינר לינוקס.

The fork pins zig 0.15.2; your macOS 26.5 is newer than that zig's bundled macOS libc stubs, so libc-linking fails at the LINK step — a tool-age vs OS-age mismatch, not the bug. Clean run = the fork's own Linux CI.

Compose freely — mitigation confirms on real hardware, Docker gives the clean green/red, the PR fixes the root. All three are independent.
