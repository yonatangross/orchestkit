🛡️ Dependency-Confusion Guard

Blue Bear Security flagged npm install @anthropic-ai/claude in OrchestKit docs — an unclaimed npm name. This playground runs the same denylist the CI guard + authoring hook use. Type an install command and watch it get classified.

The trailing boundary is what makes it safe: it flags the bare package only, never a longer -suffix package. So the real CLI @anthropic-ai/claude-code is always allowed.

🚧 CI gate hard

tests/security/test-dependency-confusion.sh — offline denylist scan of authored source. Fails the build (DO NOT MERGE) if any unclaimed reference reappears. No network → no flake.

💡 Authoring hook soft

pretool/write-edit/dependency-confusion-scanner — warns (non-blocking) the moment a Write/Edit introduces an unclaimed name. Unlike the secret scanner, it does scan markdown.
