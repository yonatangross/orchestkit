What this PR adds

    

      .cmux/cmux.json declares one command, OrchestKit Dev Rig, with keywords
      orchestkit, dev, rig, plugins, build. Invoking it opens a workspace whose layout is a
      binary tree: a direction node with a split ratio and two
      children, bottoming out in pane.surfaces entries that are either a
      terminal (with an optional command) or a browser (with a
      url). Exactly one surface carries focus: true.
    

    

      Because the layout is declarative, the rig is reproducible: any teammate running the command gets
      the identical pane arrangement, working directory, and startup commands, with no manual splitting.
