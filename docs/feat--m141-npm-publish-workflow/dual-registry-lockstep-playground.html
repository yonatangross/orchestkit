Why lockstep (not independent versions)

A contract's value is that consumers reason about ONE number: @orchestkit/hook-contract@1.2.0 ⇔ orchestkit-hook-contract==1.2.0 ⇔ spec v1.2.0. The parity preflight (in BOTH publish workflows) refuses to release unless npm == PyPI == HOOK_CONTRACT_VERSION == tag — so the registries can never drift apart.

Auth: OIDC trusted publishing on both sides — no long-lived NPM_TOKEN (which can shadow gh credentials). One-time trusted-publisher setup per registry.
