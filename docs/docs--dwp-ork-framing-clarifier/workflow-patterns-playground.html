🧬 OrchestKit · Workflow Patterns Explainer

How Claude Code dynamic workflows work, the 6 patterns ork uses to shape them, and the one rule for picking the right one. ork taxonomy — not a CC standard

1 · What a dynamic workflow actually is

A workflow is a JS harness Claude writes on the fly (CC v2.1.154+, ultracode). It runs in the background and orchestrates subagents deterministically — loops, conditionals, fan-out — instead of leaving coordination to one model's improvisation. You watch it with /workflows.

// the three core primitives (CC documents these — the mechanics)
agent(prompt, {schema, model, agentType, isolation})  // one subagent → returns its result
parallel([ ()=>agent(a), ()=>agent(b) ])               // BARRIER — awaits ALL, then continues
pipeline(items, stage1, stage2)                        // NO barrier — each item flows independently

// pick model per agent · force structured output with a schema · isolate with a git worktree

⚖️ parallel vs pipeline — the choice that matters most: use parallel() only when stage N genuinely needs all of stage N-1 (dedup, early-exit, cross-item compare). Otherwise pipeline() — item A can be in stage 3 while item B is still in stage 1, so wall-clock = slowest single chain, not sum-of-slowest-per-stage.

3 · The selection rule — pick the pattern that structurally prevents your failure

Failure mode (single-context Claude)Pattern that fixes it
🎯 Goal drift — loses the objective over many turnsFan-out-synthesize (one focused goal per agent)
🪞 Self-preferential bias — favors its own work when judgingAdversarial verification (blind refuter)
😴 Agentic laziness — declares done on partial progressLoop-until-done + /goal
📊 Hard-to-score — taste/ranking degrades at scaleTournament (pairwise)

4 · A real run (this very session)

The verification that produced this PR was a workflow — 4 verifiers in parallel() then an adversarial critic. It dogfooded pattern #2 (the critic caught a mis-versioned release-notes draft no single agent flagged).

phase('Verify')
const [align, quality, integ, notes] = await parallel([
  ()=> agent('verify the patterns doc vs current CC', {agentType:'claude-code-guide'}),
  ()=> agent('quality-review the 2 new files',         {agentType:'Explore'}),
  ()=> agent('integration + status sweep (git/gh)',    {agentType:'general-purpose'}),
  ()=> agent('draft release notes 8.22.0..HEAD',       {agentType:'general-purpose'}),
])
phase('Critique')
const critic = await agent(`find what the 4 reports MISSED: ${JSON.stringify({align,quality,integ,notes})}`)
// → critic caught: notes were titled 8.22.0 but the real delta is 8.23.0 ✅
